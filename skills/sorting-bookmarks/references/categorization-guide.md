# Руководство по категоризации закладок

## Принципы анализа

Для каждой закладки из `bookmarks_analysis.json`:

1. **Читай `crawled_title` и `description`** — это основные источники информации о содержимом страницы
2. **Если `success: false`** — ориентируйся на `original_title` и URL
3. **Определи тему** — технология, сфера деятельности, тип контента
4. **Группируй по смыслу**, а не по источнику (не "Все с Habr", а "AI: RAG-системы")

## Правила формирования категорий

### Размер категории
- **Оптимально:** 3-15 ссылок в категории
- **Если >20 ссылок:** разбей на подкатегории (например, "AI" -> "AI: Инструменты", "AI: RAG-системы", "AI: Генерация контента")
- **Если <3 ссылки:** рассмотри объединение с родственной категорией
- **Если 1 ссылка:** присоедини к ближайшей по теме категории или к "Прочее"

### Именование категорий
- Используй язык, на котором написано большинство закладок
- Краткие, понятные названия
- Для специфичных технических тем можно использовать англоязычные термины: "SEO и контент", "Claude Code и MCP"
- Для личных/нетехнических — описательные названия: "[Для работы]", "Образование и саморазвитие"

### Специальные категории
- **"Прочее"** — для ссылок, не вписывающихся ни в одну категорию (минимизируй эту группу)
- Папки в квадратных скобках `[Имя]` — пользовательские метки, сохраняй если уместно

## Формат выходного файла

Создай файл `categories_mapping.json` со следующей структурой:

```json
{
  "categories": {
    "Название категории 1": [
      "https://example.com/url1",
      "https://example.com/url2"
    ],
    "Название категории 2": [
      "https://example.com/url3"
    ]
  }
}
```

### Требования к маппингу
- **Каждый URL** из анализа должен быть в ровно одной категории
- **Нет дубликатов** — один URL не может быть в нескольких категориях
- **Нет пустых категорий** — каждая категория должна содержать хотя бы 1 URL
- **URL должны совпадать точно** с теми, что в `bookmarks_analysis.json`

## Пример сокращённого маппинга

```json
{
  "categories": {
    "AI: Генерация контента": [
      "https://app.reve.com/",
      "https://habr.com/ru/articles/955602/",
      "https://www.minimax.io/audio/voices-cloning"
    ],
    "AI: RAG-системы": [
      "https://habr.com/ru/articles/969740/",
      "https://habr.com/ru/articles/974308/",
      "https://thecode.media/ragsistema-dannyye/"
    ],
    "Claude Code и MCP": [
      "https://github.com/wshobson/agents",
      "https://habr.com/ru/articles/971066/",
      "https://anthropic.skilljar.com/claude-code-in-action"
    ],
    "Прочее": [
      "https://www.gismeteo.ru/news/animals/..."
    ]
  }
}
```

## Чеклист перед финализацией

- [ ] Все URL из анализа распределены по категориям
- [ ] Нет дубликатов URL между категориями
- [ ] Нет пустых категорий
- [ ] Размер каждой категории в пределах 3-15 (допустимы исключения с обоснованием)
- [ ] Названия категорий понятны и последовательны
- [ ] JSON валиден
